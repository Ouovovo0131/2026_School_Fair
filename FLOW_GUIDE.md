# 🎮 雙流程系統指南

## 功能概述

系統現已支持兩個獨立流程，讓用戶在登入後自由選擇：

### 1️⃣ **玩遊戲流程** 🎯
- 完成校園各處的任務
- 蒐集徽章和獎品
- 查看進度和排名
- **功能**：
  - 問答題挑戰
  - 照片上傳
  - 進度追蹤
  - 獎品兌換
  - 📍 **新增**：內置地圖查看

### 2️⃣ **查看地圖流程** 🗺️
- 瀏覽園遊會全部攤位
- 了解攤位專區分佈
- 查看各攤位詳細說明
- 不需要完成任務

## 攤位數據修改方法

### 編輯 Map.tsx 中的攤位列表

**位置**：`app/components/Map.tsx` 第 10-17 行

```typescript
const stalls = [
  { id: 1, name: "攤位名稱", location: "位置", icon: "🍱", description: "描述", highlight: "標籤" },
  // ... 更多攤位
];
```

### 新增攤位步驟

1. 在 `stalls` 陣列中添加新對象
2. 在 `zones` 陣列中添加新區域（如需要）
3. 更新區域的 `stallIds` 以包含新攤位 ID

**示例**：
```typescript
// 新增攤位
{ id: 9, name: "新攤位", location: "新位置", icon: "🎪", description: "新描述", highlight: "新標籤" }

// 新增區域
{ id: 9, name: "新區域", color: "from-indigo-200 to-indigo-100", stallIds: [9] }
```

## 按鈕行為

### 遊戲屏幕

## 常見問題

**Q: 如何添加新攤位？**
A: 編輯 `Map.tsx` 中的 `stalls` 陣列，添加新對象並更新 `zones` 陣列。

**Q: 可以自訂攤位圖標嗎？**
A: 可以，將 `icon` 屬性改為任何 emoji 或文字。

**Q: 地圖可以在mobile上使用嗎？**
A: 可以，代碼完全響應式設計。

**Q: 攤位順序可以改嗎？**
A: 可以，直接重新排列 `stalls` 陣列中的對象。
